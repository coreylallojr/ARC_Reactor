# ◉ ARC Reactor — J.A.R.V.I.S v4

> *"Just A Rather Very Intelligent System"*

JARVIS is an open-source AI companion that lives inside [Claude Code](https://claude.ai/code). It remembers every session, learns your patterns, builds a knowledge graph of your codebase, and narrates everything with a voice — all running entirely on your machine.

---

## What's New in v4 — Full Connected Brain

| Feature | Description |
|---|---|
| **Dual-index retrieval (JFR)** | TF-IDF (< 2ms, SQLite) + vector embeddings (Ollama `nomic-embed-text`) fused via Reciprocal Rank Fusion |
| **Knowledge graph** | Force-directed D3 graph of files, skills, and relationships — click any node to hear JARVIS describe it |
| **Dynamic skill registry** | JARVIS learns your repeated patterns and proposes new skills automatically |
| **MCP server** | Expose JARVIS tools to Claude Code, Claude Desktop, and any MCP client — `jarvis_recall`, `jarvis_remember`, `jarvis_speak`, and more |
| **REST + WebSocket API** | Port 7476 — integrate from VS Code, CI, webhooks, or any HTTP client |
| **5-tab UI** | Neural · Graph · Skills · Memory · Console |
| **Activity-reactive nodes** | Each tool call lights a random subset of sphere nodes; skill activation triggers a burst animation |
| **File watcher** | Attach JARVIS to any directory — files are chunked and indexed on every save |

---

## What It Does

- **Voice narration** — Every Claude Code tool call triggers a JARVIS voice line generated by Ollama and spoken by a Piper neural TTS model trained on Paul Bettany's MCU voice
- **3D Neural UI** — Three.js sphere with 150+ Fibonacci-distributed nodes that pulse to JARVIS's voice and react to every tool call
- **Session memory** — Three layers: working (live session), episodic (days/weeks), semantic (permanent knowledge + skills)
- **Skill learning** — Observes your tool sequences across sessions; after 3 matches, proposes a named skill you can activate with a keyword
- **Knowledge graph** — Automatically extracts entities from files and conversation as JARVIS ingests them
- **MCP integration** — Any MCP client can call `jarvis_recall`, `jarvis_remember`, `jarvis_skill_run`, `jarvis_speak`, and more
- **Fully local** — No cloud calls for voice or LLM. Ollama + Piper run entirely on your machine

After installing Ollama, pull both models:
```bash
ollama pull llama3.2:1b          # voice narration
ollama pull nomic-embed-text     # knowledge embeddings (v4)
```

## JARVIS Voice Model

JARVIS uses the [jgkawell/jarvis](https://github.com/jgkawell/jarvis) Piper ONNX model — trained specifically on Paul Bettany's MCU JARVIS voice.

1. Download `jarvis-medium.onnx` and `jarvis-medium.onnx.json` from the [releases page](https://github.com/jgkawell/jarvis/releases)
2. Place both files in: `~/.claude/jarvis-piper/`

```
~/.claude/
  jarvis-piper/
    jarvis-medium.onnx
    jarvis-medium.onnx.json
  jarvis_speak.py       ← installed automatically
  jarvis-audio/cache/   ← generated automatically
  jarvis-db/            ← SQLite knowledge base (v4)
    jarvis.db
    graph.json
```

## Installation

### Option A — Electron App (recommended)

```bash
git clone https://github.com/coreylallojr/ARC_Reactor.git
cd ARC_Reactor
npm install
npm start
```

The app opens a setup wizard on first run. It auto-detects Python, writes the Claude Code hooks, registers the JARVIS MCP server (`~/.claude/mcp.json`), and adds the `jarvis` shell alias. After setup, it lives in your system tray.

### Option B — CLI Only (no Electron)

```bash
git clone https://github.com/coreylallojr/ARC_Reactor.git
cd ARC_Reactor
node install.js
```

Then from any terminal:

```bash
jarvis start       # starts Ollama + server + opens UI
jarvis status      # check what's running
jarvis speak "Hello sir"
jarvis voice all   # narrate all tool calls (default)
jarvis voice key   # only significant tool calls
jarvis voice mute  # silence
jarvis help        # full command reference
```

## How It Works

```
Claude Code tool call
        │
PostToolUse hook fires neural-logger.js (~10ms, thin shim)
        │
named pipe → neural-daemon.js (persistent)
        │
  ┌─────┴───────────────────────────────────┐
  │  RAG retrieval (JFR dual-index)         │
  │  Pattern detection → skill proposals    │
  │  Session state + scoring                │
  │  Ollama narration + Piper TTS           │
  └─────────────────────────────────────────┘
        │
.pending-audio → SSE → Neural UI (localhost:7474)
        │
Web Audio API plays .wav → AudioContext analyser
        │
Three.js sphere nodes pulse to real FFT data
Activity-reactive nodes flash on every tool call
```

**MCP path (Claude Desktop / Claude Code MCP):**
```
MCP client → stdio → jarvis-mcp-server.js
                          │
              tools: recall, remember, skill_run,
                     graph_query, context_get, speak
```

**External tool path (VS Code, CI, webhooks):**
```
HTTP → port 7476 → jarvis-api-server.js
                         │
             REST + WebSocket /v1/events
```

## MCP Tools

Register JARVIS as an MCP server in Claude Code or Claude Desktop:

```json
{
  "mcpServers": {
    "jarvis": {
      "command": "node",
      "args": ["/path/to/ARC_Reactor/Neural/scripts/jarvis-mcp-server.js"]
    }
  }
}
```

The setup wizard writes this automatically to `~/.claude/mcp.json`.

| Tool | Description |
|---|---|
| `jarvis_recall` | Retrieve relevant memory chunks (dual-index JFR) |
| `jarvis_remember` | Store a fact, decision, or summary to long-term memory |
| `jarvis_skill_run` | Activate a named skill |
| `jarvis_graph_query` | Explore the knowledge graph from a node |
| `jarvis_context_get` | Get current session state (tool counts, active skill, focus file) |
| `jarvis_speak` | Make JARVIS say something out loud |

## REST API (port 7476)

```
GET  /v1/context          → session state
GET  /v1/skills           → all skills
POST /v1/skills           → create skill
DELETE /v1/skills/:id     → delete skill
POST /v1/recall           → { query } → top-k chunks
GET  /v1/graph            → full D3 graph JSON
POST /v1/graph/query      → { nodeId, depth } → subgraph
POST /v1/speak            → { text } → TTS
POST /v1/remember         → { content, type }
POST /v1/events/external  → ingest CI result, webhook, etc.
WS   /v1/events           → real-time event stream
```

## Configuration

All settings live in `Neural/config.json`:

| Key | Default | Description |
|---|---|---|
| `ollamaModel` | `llama3.2:1b` | Ollama model for voice generation |
| `embedModel` | `nomic-embed-text` | Ollama model for vector embeddings |
| `speakMinLevel` | `1` | `1`=all calls, `2`=key only, `99`=mute |
| `maxContextMode` | `false` | Include tool output in voice context |
| `patternThreshold` | `3` | Sessions before a pattern becomes a skill proposal |
| `pythonPath` | auto-detected | Path to Python 3 executable |

CLI shortcuts:
```bash
jarvis config set ollamaModel llama3.1:8b
jarvis config set speakMinLevel 2
jarvis config set patternThreshold 4
```

## Memory Layers

| Layer | Scope | Storage | Contents |
|---|---|---|---|
| Working | Session | `.session-state.json` | Tool counts, error streak, focus file, active skill |
| Episodic | Days/weeks | `jarvis.db` (SQLite) | Session summaries, error→fix pairs, file access patterns |
| Semantic | Permanent | `jarvis.db` + `graph.json` | Code embeddings, skill definitions, knowledge graph, user profile |
