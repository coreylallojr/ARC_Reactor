# ◉ ARC Reactor — J.A.R.V.I.S

> *"Just A Rather Very Intelligent System"*

JARVIS is an AI agent companion that lives inside [Claude Code](https://claude.ai/code). It narrates every action your AI agent takes with a voice, renders a live 3D visualization that pulses with the audio waveform, and gives you a single command to manage everything.

---

## What It Does

- **Voice narration** — Every Claude Code tool call triggers a JARVIS voice line generated by a local LLM (Ollama) and spoken by a Piper neural TTS model trained on Paul Bettany's MCU voice
- **3D Neural UI** — Three.js sphere with 150+ nodes, each mapped to a frequency bin of the audio — the sphere literally expands and pulses to JARVIS's voice in real time
- **Fully local** — No cloud calls for voice or LLM. Ollama + Piper run entirely on your machine
- **Single command** — `jarvis start` launches everything from any terminal
- **Persistent** — Hooks into Claude Code globally, active in every session automatically

## Prerequisites

| Requirement | Install |
|---|---|
| [Node.js](https://nodejs.org) v18+ | Required to run the server and CLI |
| [Python 3.x](https://python.org) | Required for Piper TTS |
| [Ollama](https://ollama.com) | Free local LLM — generates the JARVIS voice lines |
| [Claude Code](https://claude.ai/code) | The AI agent JARVIS rides inside |
| piper-tts | `pip install piper-tts` |

After installing Ollama, pull the voice model:
```bash
ollama pull llama3.2:1b
```

## JARVIS Voice Model

JARVIS uses the [jgkawell/jarvis](https://github.com/jgkawell/jarvis) Piper ONNX model — trained specifically on Paul Bettany's MCU JARVIS voice.

1. Download `jarvis-medium.onnx` and `jarvis-medium.onnx.json` from the [releases page](https://github.com/jgkawell/jarvis/releases)
2. Place both files in: `~/.claude/jarvis-piper/`

```
~/.claude/
  jarvis-piper/
    jarvis-medium.onnx
    jarvis-medium.onnx.json
  jarvis_speak.py       <- installed automatically
  jarvis-audio/cache/   <- generated automatically
```

## Installation

### Option A — Electron App (recommended)

```bash
git clone https://github.com/coreylallojr/ARC_Reactor.git
cd ARC_Reactor
npm install
npm start
```

The app opens a setup wizard on first run. It auto-detects Python, writes the Claude Code hooks, and adds the `jarvis` shell alias. After setup, it lives in your system tray.

### Option B — CLI Only (no Electron)

```bash
git clone https://github.com/coreylallojr/ARC_Reactor.git
cd ARC_Reactor
node install.js
```

Then from any terminal:

```bash
jarvis start       # starts Ollama + server + opens UI
jarvis status      # check what's running
jarvis speak "Hello sir"
jarvis voice all   # narrate all tool calls (default)
jarvis voice key   # only significant tool calls
jarvis voice mute  # silence
jarvis help        # full command reference
```

## How It Works

```
Claude Code tool call
        |
PostToolUse hook fires neural-logger.js
        |
Scores significance -> prompts Ollama -> gets JARVIS line
        |
jarvis_speak.py -> Piper ONNX -> .wav file cached
        |
filename written to .pending-audio
        |
Neural UI (localhost:7474) polls every 150ms
        |
Web Audio API plays .wav -> AudioContext analyser
        |
Every frame: sphere nodes + bloom + rings react to real FFT data
```

## Configuration

All settings live in `Neural/config.json`:

| Key | Default | Description |
|---|---|---|
| `ollamaModel` | `llama3.2:1b` | Ollama model for voice generation |
| `speakMinLevel` | `1` | `1`=all calls, `2`=key only, `99`=mute |
| `maxContextMode` | `false` | Include tool output in voice context |
| `pythonPath` | auto-detected | Path to Python 3 executable |

Or use the CLI:
```bash
jarvis config set ollamaModel llama3.1:8b
jarvis config set speakMinLevel 2
```

## Architecture

```
ARC_Reactor/
├── main.js                     <- Electron: tray, server lifecycle, setup
├── preload.js                  <- IPC bridge for setup window
├── install.js                  <- CLI-only installer (no Electron required)
├── setup/setup.html            <- First-run setup wizard UI
├── Neural/
│   ├── config.json             <- All configuration
│   ├── scripts/
│   │   ├── jarvis-cli.js       <- `jarvis` command
│   │   ├── neural-logger.js    <- Claude Code hook handler + voice generation
│   │   └── neural-ui-server.js <- HTTP server :7474, audio bridge
│   └── ui/
│       └── jarvis-neural.html  <- Three.js sphere + Web Audio UI
└── python/
    └── jarvis_speak.py         <- Piper TTS wrapper (cross-platform)
```
